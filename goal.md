# Current goal

**Set:** 2026-07-15
**Status:** in progress

Confirmed via a HelmControl New Goal intake chat ("Start This Goal"): a one-to-one Trello clone on Supabase for a specific known organization (not public), with multi-user accounts and permissions layered on Supabase auth. Boards support invites by username or email, attachments of any file type (images, scripts, zips) plus links, and custom backgrounds (color picker or photo). Cards show a creation timestamp, an image preview/cover even when no card is selected (auto-picks the most recent image, with a manual "make cover" override), and a visual-only completion checkmark that doesn't drive other backend behavior. There's an in-app notifications section for invites and other events (no email notifications), plus fixes for a drag-and-drop visibility glitch and a white-rectangle rendering bug on horizontal scroll. The newest addition is a calendar view of cards by due date, combined with full two-way Google Calendar sync via OAuth for each user's own Google account — since this is limited to the organization's known members, the Google OAuth app can run in "testing" mode with everyone added as test users, avoiding Google's public verification process.

## Done when
- Multi-user accounts and permissions work on top of Supabase auth: board creators can invite others by username or email, and non-owner members are correctly gated from owner-only actions.
- Boards support fully customizable backgrounds via both a color picker and an uploaded photo.
- Cards accept attachments of any file type plus links; image attachments show an inline thumbnail preview.
- An in-app notification section surfaces invites and other board events (no email delivery).
- Cards display their creation timestamp.
- Card fronts show a cover image without opening the card — defaulting to the most recently added image attachment, with a manual "make cover" override that persists.
- Cards have a visual-only completion checkmark on the card front that toggles and persists without driving any other backend behavior (no filtering/automation/notifications tied to it).
- Dragging a card between lists never makes it disappear mid-drag; scrolling the board as far right as it goes never reveals a stray white (or any other non-background-colored) rectangle.
- Cards have a start-date field (optionally a plain deadline with no end time, or a real meeting with an end time) and a calendar view that shows cards laid out by date, with a time range shown for meeting-style cards.
- Each user can connect their own Google account via OAuth and get full two-way sync between their cards' dates and their Google Calendar: card date changes propagate to Google Calendar as either an all-day event (deadline-style) or a timed event with a real start/end (meeting-style), and changes made on the Google Calendar side propagate back onto the card, correctly distinguishing all-day from timed events.
- The Google OAuth consent screen runs in "testing" publish status with organization members added as test users — no public verification flow required.
- All of the above verified live in a real browser against the live Supabase backend (and a real connected Google account for the calendar sync), not just via unit/type checks.
