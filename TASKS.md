Also fixed a real bug found in the process (see below). Automated direct-query spec (tests/e2e/rls/rls-direct-query.spec.ts) still needs the npm-install unblock to run as part of CI.
  - **Bug found and fixed during this verification**: board creation was failing 100% of the time with "new row violates row-level security policy for table boards". Root cause: the `board_members` row that makes the creator a member (needed for the `boards` SELECT policy) is inserted by an `AFTER INSERT` trigger, which fires *after* PostgREST evaluates the `RETURNING` clause that `.insert().select()` relies on — so the creator's own just-created board failed its own read-back. Fixed in `supabase/migrations/20260714120010_fix_board_owner_select_race.sql` by letting `owner_id = auth.uid()` satisfy the SELECT policy directly, independent of the trigger's timing.

  - Also disabled Supabase's default "require email confirmation before login" (`supabase config push`), per user request — this is an internal tool, not public-facing.
